**Join the Elite Network of Geospatial Engineers**

## 🎖️ Operational Protocol

By participating in this project, you agree to maintain the highest standards of code quality and operational security.

### 📡 Reporting Intelligence Gaps (Bugs)
- Utilize the GitHub Issue Tracker with the `[INCIDENT]` prefix.
- Provide a full sitrep: environment, steps to reproduce, and expected vs. actual signal output.
- Attach tactical data (logs, screenshots) to accelerate resolution.

### 🚀 Proposing Strategic Enhancements
- Feature requests should be submitted as `[STRATEGY]` issues.
- Explain the tactical advantage this enhancement provides to the mission.

### 🛠️ Extraction & Fusion (Pull Requests)
1. Fork the mission repository and create a feature branch (`feat/`, `fix/`, `refactor/`).
2. Maintain signal integrity: ensure all neural models and API endpoints are tested.
3. Adhere to the **Tactical Style Guide** (see below).
4. Submit your PR with a clear executive summary of changes.

## 💻 Technical Setup

### Neural Backend
1. `cd backend`
2. `pip install -r requirements.txt`
3. `python main.py --start-command-center`

### Tactical Frontend
1. `cd frontend`
2. `npm install`
3. `npm run dev`

---

## 🎨 Tactical Style Guide

- **TypeScript**: Mandatory for all frontend components. Zero `any` tolerance.
- **Python**: PEP 8 compliance with type hinting.
- **Design**: All UI components must adhere to the **EarthMind Design System** (Glassmorphism, CSS variables, Tailwind v4).
- **Documentation**: Code should be self-documenting; complex logic requires tactical comments.
